Dim DataSource

PV syntax

The standard prefix for the datasource is dim:// which can be omitted if configured as the default datasource.

SyntaxDescription
dim://PV

Client side parameters

The data source supports a small number of client parameters that specify how to handle the topic at the client side. These parameters are stripped from the request and never reach the server.

The parameter block is a JSON object separated from the pv name by a space (i.e ' '). The currently supported parameters are:

  • dim://mypv {"topic":"NameOfTopic"}
    changes the default topic from preferences
Note that at this time the JSON notation has to match exactly (no extra spaces) and does not support the mix of parameters. This will be fixed in a future version. Note PVManager will think this is a formula because of the ":", so single quotes are needed in this case: 'dim://mypv {"topic":"NameOfTopic"}'
